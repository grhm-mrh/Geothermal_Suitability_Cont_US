

## 📌 Project Objective
To identify highly viable geothermal target zones across the contiguous United States by extracting continuous subsurface temperature data from static legacy maps, and intersecting it with active Quaternary faults, Holocene volcanoes, and legal land-use constraints. Finally, to automate the generation of natural-language site assessments for LLM evaluation using PyQGIS.

## 🛠️ The Spatial Data Engineering Workflow

This project required a complete data cleaning pipeline—reverse-engineering a heavily compressed, un-legend-matched JPEG image into a queryable, scientifically continuous geospatial database.

### Phase 1: Spatial Rectification & The Bounding Box Fix
*   **Georeferencing:** Imported a legacy SMU Geothermal JPEG. Corrected a projection mismatch by reprojecting the workspace to **EPSG:5070 (NAD83 / Conus Albers)** to guarantee mathematically accurate Euclidean distance calculations in meters.
*   **Bounding Box Isolation:** Created a "Single parts" polygon mask of the contiguous Lower 48 states. Clipped the raw map using this mask while explicitly setting the NoData value to `0`, successfully stripping away the oceans, Canada, and the mathematical bounding box trap.

### Phase 2: Color Quantization & Vector Burning (Anomaly Patching)
*   **Unsupervised Clustering:** Utilized `RGB to PCT` quantization to force millions of RGB gradients into discrete geological color IDs. 
*   **Overcoming Algorithmic Limits:** Unsupervised clustering naturally eliminated critical high-heat micro-anomalies (e.g., SW Idaho and SoCal hotspots) due to statistical weighting.
*   **Vector Burning:** Bypassed this algorithmic failure by manually digitizing the missing micro-anomalies into a new vector shapefile. Rasterized these polygons and used conditional Raster Calculator logic to mathematically "stamp" these anomalies perfectly onto the base raster.

### Phase 3: Noise Eradication & Morphological Fill
*   **The Cheat Sheet:** Mapped the random PCT ID numbers to their true real-world temperatures (e.g., ID 17 = 150°C) using the map's legend.
*   **Noise Vaporization:** Ran `Reclassify by Table` using only the verified IDs. By enabling the "Use NoData" function, instantly vaporized all state borders, black text, and JPEG compression artifacts into transparent holes. 
*   **Data Interpolation:** Deployed a 500-pixel **Morphological Fill (IDW Interpolation)** to bridge the massive data gaps left by the deleted logos, restoring the map to a continuous geological surface.
*   **Decimal Snapping:** Ran a final Range Reclassification (`min <= value < max`) to mathematically snap all floating-point decimals generated by the fill tool back to discrete temperature integers.

### Phase 4: Multi-Criteria Decision Analysis (MCDA)
*   **Permeability & Heat Sources:** Sourced and reprojected USGS Quaternary Faults and Smithsonian Holocene Volcanoes to EPSG:5070. Rasterized the vectors and ran `Proximity (Raster Distance)` to generate continuous Euclidean distance grids.
*   **Legal Exclusion Mask:** Rasterized a Protected Lands shapefile into a Boolean mask (`0` = Illegal to drill/National Parks, `1` = Legal to drill).
*   **The Final Matrix Math:** Executed the final MCDA formula in the QGIS Raster Calculator:

> `("Temperature_Raster" + ("Distance_to_Faults" <= 15000) * 50 + ("Distance_to_Volcanoes" <= 50000) * 50) * "Legal_Mask"`

**Logic:** *If the subsurface temperature is highly viable, AND the site is within 15km of an active fault (natural permeability), AND within 50km of an active volcano (heat source), the model assigns a compounding priority bonus. If the site falls within protected lands, the Boolean mask multiplies the entire score by 0, instantly disqualifying it.*

### Phase 5: PyQGIS AI Integration
*   Isolated the top-scoring target zones and converted the pixel clusters into discrete vector polygons.
*   Utilized **Zonal Statistics** to extract multi-depth temperatures and exact fault distances into the attribute table.
*   Engineered a **PyQGIS automation script** that loops through the geodatabase, evaluates the spatial metrics, and automatically generates highly structured, natural-language prompts for frontier AI (LLM) site evaluation.
